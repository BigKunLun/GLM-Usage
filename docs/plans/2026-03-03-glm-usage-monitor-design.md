# GLM Usage Monitor - 设计文档

## 概述

macOS菜单栏应用，用于快速查看GLM Coding Plan的用量信息，无需打开浏览器。

## 目标用户

已订阅GLM Coding Plan的用户，希望在macOS菜单栏快速查看用量。

## 功能需求

### 核心功能
- 显示三种额度使用情况：
  1. 每5小时使用额度
  2. 每周使用额度
  3. MCP每月额度
- 手动刷新数据
- 显示上次更新时间

### 数据获取
- 读取Dia浏览器（Chrome系）的Cookie
- 使用Cookie调用GLM API获取用量数据
- 无需用户单独登录

## 技术方案

### 技术栈
- **语言:** Swift
- **UI框架:** SwiftUI
- **最低系统:** macOS 13.0 (Ventura)
- **菜单栏:** MenuBarExtra (原生API)

## 数据模型

```swift
struct GLMUsage {
    let hourly5: Quota      // 每5小时额度
    let weekly: Quota       // 每周额度
    let mcpMonthly: Quota   // MCP每月额度
    let lastUpdated: Date   // 更新时间
}

struct Quota {
    let used: Int           // 已使用
    let total: Int          // 总额度
    var percentage: Double  // 使用百分比
}
```

## Cookie读取流程

1. 定位Dia浏览器Cookie文件
   - 路径: `~/Library/Application Support/Dia/Cookies`

2. 读取SQLite数据库
   - 查询 bigmodel.cn 域名的Cookie
   - 提取关键Cookie值

3. 解密加密的Cookie值
   - Chrome系浏览器使用DPAPI加密
   - 调用macOS Keychain API解密

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| Dia浏览器未安装 | 显示提示："请确保已安装Dia浏览器并登录GLM" |
| Cookie已过期 | 显示提示："登录已过期，请在浏览器重新登录" |
| 网络请求失败 | 显示错误信息，保留上次数据，提供重试按钮 |
| API返回异常 | 显示"数据解析失败" |
| 无权限访问Keychain | 引导用户在系统设置中授权 |

## 权限需求

- 访问用户文件（读取Cookie数据库）
- 访问Keychain（解密Cookie）
- 网络访问（调用API）
